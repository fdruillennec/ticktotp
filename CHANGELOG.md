## Application Screenshots

### 1. Initial Screen - Email Not Registered
![Email Not Registered Screen](screenshots/1.png)

The initial screen when a user's email is not yet registered in the system. Users can:
- Connect an authenticator app to register
- Return to the home screen

### 2. QR Code Pairing
![QR Code Pairing Screen](screenshots/2.png)

After clicking "Connect an Authenticator App", users are presented with a QR code to scan with their authenticator app (Google Authenticator, Authy, etc.). This establishes the TOTP connection.

### 3. TOTP Code Entry
![TOTP Code Entry Screen](screenshots/3.png)

Users enter their 6-digit TOTP code generated by their authenticator app. The interface features:
- Six individual input boxes for each digit
- Visual focus indicator
- Verify button to submit the code
- Back to Home option

### 4. Login Screen
![Login Screen](screenshots/4.png)

The main login interface where registered users enter their email address to begin the TOTP authentication process.

### 5. Successful Connection
![Success Screen](screenshots/5.png)

Confirmation screen displayed after successful TOTP verification, showing:
- User's connected email address
- Green confirmation message
- Option to return to home

## User Flow

1. **Initial Access** → User enters email on login screen
2. **Registration Check** → System checks if email is registered
3. **First Time Users** → Shown QR code to pair authenticator app
4. **TOTP Setup** → User scans QR code with authenticator app
5. **Code Entry** → User enters 6-digit TOTP code
6. **Verification** → System validates the TOTP code
7. **Success** → User is authenticated and connected

## Technical Stack

- **Frontend**: React with Vite and Tailwind CSS
- **Backend**: Go API
- **Storage**: Redis for TOTP secrets and session management
- **Deployment**: Docker Compose
